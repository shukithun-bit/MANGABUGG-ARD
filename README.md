# MangabuffCard Telegram Bot

Коллекционный Telegram gacha-бот на Python/aiogram 3.

## Что есть в этой версии

- Вселенные: игрок выбирает вселенную при регистрации и получает карты только из неё.
- Админ может создавать, скрывать, открывать и удалять вселенные.
- Админ/модератор может добавлять карты в выбранную вселенную.
- Карты поддерживают фото и GIF.
- Для каждой карты отдельно задаются осколки за раскол и очки опыта.
- Коллекция на русском, с разделением по редкости и счётчиками `есть/всего`.
- Смена вселенной платная: карты старой вселенной раскалываются в осколки.
- Магазин создаётся через бота; товары можно скрывать, открывать и удалять.
- Создатель может дарить крутки по ID, @тегу или игровому нику.
- Игрок может менять игровой ник через `/nick`.
- В топе показываются только игровые ники и очки.
- Создатель может выдавать/забирать роль модератора.
- Система сезонов: старт сезона, завершение сезона, уведомление игрокам, топ-100, сброс прогресса.
- Настройки цен через бота, включая цену смены вселенной.

## Railway

Variables:

```env
BOT_TOKEN=токен_бота
ADMIN_IDS=123456789,987654321
TZ=Europe/Moscow
DB_PATH=/data/game.db
DROP_COOLDOWN_MINUTES=30
START_POINTS=0
UNIVERSE_CHANGE_COST=500
CARD_IMAGE_DIR=/data/images/cards
```

Если используешь SQLite на Railway, подключи Volume и укажи `/data` для `DB_PATH` и `CARD_IMAGE_DIR`.

## Запуск

```bash
pip install -r requirements.txt
python bot.py
```

Для Railway используется `Procfile`:

```text
worker: python bot.py
```

## Команды

Игроки:

- `/start`, `/menu`
- `/drop`
- `/daily`
- `/collection`
- `/shop`
- `/profile`
- `/top`
- `/universe`
- `/nick НовыйНик`

Создатель:

- `/admin`
- `/addcard`

Модератор:

- `/addcard`

## Важно про медиа и кнопки

Большинство inline-кнопок редактируют старое сообщение. При выдаче карты с фото/GIF Telegram не всегда позволяет превратить текстовое меню в медиа-сообщение, поэтому бот удаляет старое меню и отправляет карточку с изображением/гифкой вместо него.
